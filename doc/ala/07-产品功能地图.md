## 用户旅程

### 新用户首次使用

```
1. npm install -g happy-coder
2. 终端运行 happy → 显示 QR 码
3. 下载手机 App → 扫码配对
4. 配对成功 → 终端开始 AI 会话
5. 手机实时看到会话内容
6. 可以在手机上发送指令、查看文件、审批操作
```

### 远程使用场景

```
1. 开发者在公司电脑运行 happy daemon（后台常驻）
2. 下班后用手机打开 App
3. 在手机上创建新会话 → "帮我 review 今天的 PR"
4. 公司电脑的 Daemon 自动启动 Claude Code
5. 手机实时收到 review 结果 + 推送通知
```

## 关键产品指标建议

| 指标 | 说明 |
|------|------|
| DAU | 日活跃用户数 |
| 会话数/用户/日 | 人均会话频次 |
| 远程 vs 本地会话比 | 远程控制功能渗透率 |
| 设备配对成功率 | QR 码认证转化率 |
| 消息延迟 | 端到端消息延迟（含加密） |
| 留存率 | D1/D7/D30 留存 |
| 多 Agent 使用分布 | Claude vs Codex vs Gemini 占比 |
| 付费转化率 | 免费 → 付费转化 |

## 竞争格局

```
┌────────────────────────────────────────────────────┐
│                  AI 编程工具生态                     │
├────────────┬───────────────────────────────────────┤
│ IDE 集成   │ GitHub Copilot, Cursor, Windsurf      │
├────────────┼───────────────────────────────────────┤
│ CLI Agent  │ Claude Code, Codex CLI, Aider         │
├────────────┼───────────────────────────────────────┤
│ 远程控制   │ ★ Happy Coder（独占位）               │
├────────────┼───────────────────────────────────────┤
│ 终端增强   │ Warp, Fig (已收购)                    │
└────────────┴───────────────────────────────────────┘

Happy 的独特定位：不与 AI Agent 竞争，而是做 Agent 的「移动遥控器」
```

## 产品发展方向（建议思考）

| 方向 | 描述 |
|------|------|
| **团队协作** | 多人共享会话、团队 Agent 管理 |
| **Agent 市场** | 接入更多 AI Agent，变成通用远程控制平台 |
| **工作流自动化** | 定时任务、触发器、CI 集成 |
| **企业版** | 自托管、SSO、审计日志、合规 |
| **会话模板** | 预设 Prompt、项目配置快速启动 |
| **深度 GitHub 集成** | PR Review、Issue 处理、代码搜索 |
| **AI 成本管理** | Token 用量统计、预算告警 |
| **离线模式** | 断网后消息缓存、重连同步 |
