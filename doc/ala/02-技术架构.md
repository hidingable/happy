## Monorepo 结构

```
happy/
├── packages/
│   ├── happy-app/        # 移动端 + Web + 桌面端客户端（Expo）
│   ├── happy-cli/        # CLI 工具（npm: happy-coder）
│   ├── happy-server/     # 后端服务（Fastify）
│   ├── happy-wire/       # 共享 Zod Schema / 协议类型
│   ├── happy-agent/      # 远程 Agent 控制 CLI
│   └── happy-app-logs/   # 开发调试日志服务
├── docs/                 # 内部文档
├── environments/         # 开发环境管理
├── scripts/              # 发布脚本
├── Dockerfile            # 独立模式（PGlite）
├── Dockerfile.server     # 生产模式（外部 PG + Redis）
└── Dockerfile.webapp     # Web 应用（Nginx）
```

## 数据流架构

### 会话创建与通信

```
手机 App                    Server                    CLI Daemon
  │                          │                          │
  │── 创建会话（加密）───────→│                          │
  │                          │── Socket 推送 ──────────→│
  │                          │                          │── 启动 Claude/Codex
  │                          │                          │
  │                          │←── 实时消息（加密）───────│
  │←── Socket 推送 ──────────│                          │
  │                          │                          │
  │── 发送指令（加密）───────→│── 转发（加密）──────────→│
  │                          │                          │── Agent 执行
```

### 加密架构

```
┌────────────────────────────────────────────┐
│              加密层级                        │
├────────────────────────────────────────────┤
│ L1: 用户主密钥 (Master Key)                 │
│     └─ 从账户密钥对派生                      │
│                                            │
│ L2: 实体加密密钥 (Data Encryption Key)      │
│     └─ 每个 Session/Machine/Artifact 独立   │
│     └─ 用 Master Key 加密后存储到 Server    │
│                                            │
│ L3: 消息加密                                │
│     └─ 用实体 DEK 加密消息内容               │
│     └─ Server 只存储密文，永远看不到明文     │
└────────────────────────────────────────────┘
```

## 部署架构

| 部署模式 | 说明 | Docker |
|---------|------|--------|
| **独立模式** | 内嵌 PGlite，零外部依赖 | `Dockerfile` → port 3005 |
| **生产模式** | 外部 PostgreSQL + Redis + MinIO | `Dockerfile.server` → port 3000 |
| **Web 应用** | Expo 静态导出 + Nginx | `Dockerfile.webapp` → port 80 |
| **移动端** | EAS Build → App Store / Google Play | EAS 云构建 |

## 关键设计决策

1. **端到端加密优先** — 服务器是"加密中继"，不能解密任何用户数据
2. **双模式 CLI** — 本地终端模式 + 远程 SDK 模式，可实时切换
3. **多 Agent 支持** — Claude Code / Codex / Gemini / OpenClaw 可互换
4. **后台 Daemon** — 手机可以远程创建会话，无需开发者在电脑前操作
5. **InvalidateSync 模式** — 自研同步协调模式，管理多数据源的 fetch/retry
6. **文件路由** — Expo Router v6 文件系统路由，降低路由管理心智负担
