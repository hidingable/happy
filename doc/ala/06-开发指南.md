# Happy Coder 开发指南

## 环境要求

| 工具 | 版本 |
|------|------|
| Node.js | 20+ |
| Yarn | 1.22+（Classic） |
| Expo CLI | 随 Expo SDK 54 |
| Xcode | iOS 开发 |
| Android Studio | Android 开发 |
| Docker | 服务端部署（可选） |

## 首次设置

```bash
# 1. 安装依赖
yarn install

# 2. 启动开发环境（推荐使用环境管理系统）
yarn env:new my-dev          # 创建隔离开发环境
yarn env:use my-dev          # 切换到该环境

# 3. 启动服务
yarn env:server              # 启动后端（standalone 模式，内嵌 PGlite）
yarn env:web                 # 启动 Web 版
yarn env:ios                 # 启动 iOS 模拟器
yarn env:android             # 启动 Android 模拟器
yarn env:cli                 # 启动 CLI 开发模式
```

## 常用开发命令

```bash
# 根目录 scripts
yarn cli                     # 运行 CLI 开发模式
yarn web                     # 启动 Web 开发服务器
yarn app-logs                # 启动日志查看器
yarn release                 # 交互式发布

# 环境管理
yarn env:list                # 列出所有环境
yarn env:current             # 查看当前环境
yarn env:seed                # 填充测试数据
yarn env:up                  # 启动环境
yarn env:down                # 停止环境
```

## 包级别开发

### happy-app（移动端/Web）

```bash
cd packages/happy-app

# 类型检查
yarn typecheck

# 启动开发
npx expo start               # 通用启动
npx expo start --ios          # iOS
npx expo start --android      # Android
npx expo start --web          # Web

# 构建
eas build --profile development --platform ios
eas build --profile development --platform android

# OTA 更新
eas update --branch production
```

### happy-cli（CLI）

```bash
cd packages/happy-cli

# 开发模式
yarn cli

# 构建
yarn build                    # pkgroll 构建

# 测试
yarn test                     # Vitest
```

### happy-server（后端）

```bash
cd packages/happy-server

# 开发模式（standalone，内嵌 PGlite）
yarn standalone

# 生产模式（需要外部 PostgreSQL + Redis）
yarn start

# 数据库
npx prisma migrate dev        # 运行迁移
npx prisma generate           # 生成客户端
npx prisma studio             # 打开数据库管理界面
```

### happy-wire（共享类型）

```bash
cd packages/happy-wire
yarn build                    # pkgroll 构建
```

## 项目约定

### 代码风格

- TypeScript strict 模式
- Zod 做运行时类型校验
- 路径别名: `@/*` → `sources/*`（happy-app）

### App 开发约定

（详见 `packages/happy-app/CLAUDE.md`）

- **路由**: Expo Router 文件系统路由，新页面直接在 `sources/app/` 下创建文件
- **样式**: 使用 `react-native-unistyles` 的 `createStyleSheet`，不用内联样式
- **i18n**: 所有用户可见文本通过 `useText()` Hook 获取
- **组件**: 优先使用 `Item`/`ItemGroup`/`ItemList` 等基础组件
- **状态**: 全局状态用 Zustand（`storage.ts`），局部状态用 `useState`

### Server 开发约定

（详见 `packages/happy-server/CLAUDE.md`）

- **路由**: Fastify + Zod Schema 验证
- **数据库**: Prisma ORM，不写原生 SQL
- **模块系统**: 功能按模块组织在 `sources/app/` 下

### CLI 开发约定

（详见 `packages/happy-cli/CLAUDE.md`）

- **UI**: 使用 Ink（React for CLI）渲染终端界面
- **加密**: 所有传输数据使用 TweetNaCl 加密

## 部署

### Docker 部署

```bash
# 独立模式（零依赖，适合个人/小团队）
docker build -t happy-standalone -f Dockerfile .
docker run -p 3005:3005 happy-standalone

# 生产模式（需配置 PG + Redis + MinIO）
docker build -t happy-server -f Dockerfile.server .

# Web 应用
docker build -t happy-webapp -f Dockerfile.webapp .
```

### 移动端发布

```bash
# EAS Build
eas build --profile production --platform ios
eas build --profile production --platform android

# 提交到商店
eas submit --platform ios
eas submit --platform android

# OTA 更新（无需重新提交商店）
eas update --branch production
```

### CLI 发布

```bash
yarn release                  # 交互式选择要发布的包
# 发布到 npm: happy-coder
```

## 目录下的 CLAUDE.md

每个关键包都有 `CLAUDE.md`，包含针对该包的详细开发指南：

- `packages/happy-app/CLAUDE.md` — App 架构、样式、i18n、组件规范
- `packages/happy-cli/CLAUDE.md` — CLI 架构、双模式、加密、Daemon
- `packages/happy-cli/src/daemon/CLAUDE.md` — Daemon 控制流、会话管理
- `packages/happy-server/CLAUDE.md` — Server 开发模式、Fastify 模式、调试

**接手后建议先通读这四个 CLAUDE.md，是最快的深入了解途径。**
